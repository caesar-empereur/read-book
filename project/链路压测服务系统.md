## 链路压测服务系统
```
实现对全链路的微服务压测，包括单接口压测，链路压测

在应用里面引入对应的jar包，配置好生产上的中间价，数据库，缓存的落地策略
本方案采用的是影子库来实现压测请求与正常请求的数据落地区分，正确请求落到生产库，
压测请求落到生产的影子库，影子库设计的有 mysql, mongodb, redis，rabbitmq等

但是生产库与影子库的数据必须保持一致才有压测的意义，一个数据库几千万的数据跟一个数据库
是空的，插入与查询性能完全是不一样的，因此需要在生产上部署一整套跟正常库一样规模的影子库
机器性能必须保持一致，数据必须实时同步，这对小团队来说是相当有挑战的事情。
不仅仅是技术的问题，还有机器资源的问题，团队配合的问题
```
